# Improvement Proposal 05: Product Architecture Overhaul

**Date:** 2026-03-24  
**Status:** Implemented  
**Precedes:** —  

---

## Summary

Replace the monolithic Gradio application with a **separate frontend and backend**:

- **Backend:** FastAPI — model registry, versioned artifacts, inference, batch scoring, sensitivity curves, OpenAPI docs.
- **Frontend:** React + TypeScript + Vite + Tailwind — F1.com–inspired UI, dynamic forms driven by `/api/models/schema`, parameter impact charts.
- **Deployment:** Docker Compose — backend + nginx-served SPA, read-only model volume, health checks.

**No further model or dataset work** in this IP; focus is engineering and UX.

---

## Goals

1. Clear separation of concerns (UI / API / data pipeline).
2. **Model-agnostic UI:** schema and defaults from backend metadata, not hardcoded feature lists in the client.
3. **Explainability:** show how varying one input shifts `P(overtake)` (1D sensitivity + optional top perturbations).
4. **Operability:** one command to run the full stack; configurable default model via env/registry.
5. **Cleanup:** remove superseded Gradio app and CLI duplicates; relocate pipeline and notebooks; trim dead code paths.

---

## Architecture

```
Browser → React SPA (nginx :3000) → proxy /api → FastAPI (:8000) → joblib pipeline + meta JSON
                                              ↘ models/artifacts (mounted)
```

Pipeline code (`pipeline/`) remains for **offline** CSV generation only; runtime inference does not depend on FastF1.

---

## Backend (FastAPI)

| Area | Responsibility |
|------|----------------|
| `app/config.py` | `pydantic-settings`: `ARTIFACTS_DIR`, `DEFAULT_MODEL`, CORS origins |
| `app/services/model_registry.py` | Load `registry.json`, resolve `.pkl` + `_meta.json`, switch active model |
| `app/services/feature_builder.py` | Port of former `models/app.py`: single-row build, batch engineering, align columns to `meta["features"]` |
| `app/services/inference.py` | `predict_proba`, sensitivity sweep, optional local perturbation scores for “impact” |
| Routers | `/api/health`, `/api/models/*`, `/api/predict/*`, `/api/sensitivity` |

---

## Frontend (React)

| Area | Responsibility |
|------|----------------|
| Dynamic form | `GET /api/models/schema` → render sliders, selects, team badges |
| Single battle | Submit JSON → prediction card + gauge + verdict |
| Sensitivity | Select feature → line chart (Recharts) |
| Batch | Upload CSV → table + metrics + download link |
| Styling | Titillium Web, F1 palette, team colour tokens |

---

## Docker

- **backend:** Python slim, `uvicorn`, `curl` for healthcheck.
- **frontend:** multi-stage build (Node → nginx) with `/api` reverse proxy to backend service name.

---

## Repository layout (post-overhaul)

- `pipeline/` — data generation (moved from `src/pipeline/`).
- `notebooks/` — research notebooks (moved from `models/`).
- `models/artifacts/` — registry + `.pkl` + `*_meta.json` (versioned in git).
- `backend/` — FastAPI application.
- `frontend/` — Vite SPA.
- `docker-compose.yml` — root orchestration.

Removed: `models/app.py`, `models/predict.py`, `models/score_battle.py`, obsolete baseline notebook under `src/notebooks/`.
